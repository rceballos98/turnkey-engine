# Turnkey Engine

FastAPI microservice with a background worker and Postgres job queue. The agent pipeline (Firecrawl + Socrata + Claude) is a placeholder — this is the scaffolding.

## Local Development

```bash
# 1. Create virtualenv and install deps
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt

# 2. Start Postgres
docker run -d --name turnkey-pg -e POSTGRES_DB=turnkey -e POSTGRES_PASSWORD=postgres -p 5432:5432 postgres:16

# 3. Configure env
cp .env.example .env

# 4. Run migrations
alembic revision --autogenerate -m "initial tables"
alembic upgrade head

# 5. Start API
uvicorn app.main:app --reload

# 6. Start worker (separate terminal)
python -m app.worker
```

## API

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| `GET` | `/health` | No | Health check (pings DB) |
| `POST` | `/reports` | Bearer token | Create a report and enqueue for processing |
| `GET` | `/reports/{id}` | Bearer token | Get report by ID |

Auth: `Authorization: Bearer <INTERNAL_API_KEY>`

## Deploy to Render

1. Push this repo to GitHub
2. Create a new **Blueprint** in Render, point it at the repo
3. Set env vars in the Render dashboard:
   - `DATABASE_URL` — Supabase or Neon connection string
   - `ANTHROPIC_API_KEY` — Claude API key
   - `FIRECRAWL_API_KEY` — Firecrawl API key
   - `BASE_URL` — your `.onrender.com` URL
4. `INTERNAL_API_KEY` is auto-generated by Render

Architecture: API ($7/mo) + Worker ($7/mo) = ~$14/mo. Database is external (Supabase/Neon free tier).
